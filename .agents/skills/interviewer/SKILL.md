# /interviewer — 思考を引き出すインタビュー

AIがインタビュアー役となり、オーナーに質問を投げかけることで頭の中にある思考・経験・価値観を引き出す。会話のやり取りを経て、得られた内容を `thoughts/` に記録する。

## 背景

thoughts/ への入力は日記からの蒸留だけでは不十分。オーナーの頭の中に大量の未記録の思考があるが、「何から書くか」の判断コストが高く入力が進まない。AIが質問を出すことで、その判断コストをゼロにする。

## 手順

1. **テーマの決定**
   - `$ARGUMENTS` でテーマが指定されていればそれを使う
   - 指定がなければ、以下を参考にテーマを提案する:
     - `profile/about.md` の関心・価値観から深掘りできそうなもの
     - `mcp__life-rag__search(query="テーマ候補", layer="thoughts")` で既存の蓄積度を確認し、まだ深掘りされていない領域を優先する
     - `profile/goals.md` に関連するがまだ言語化されていないもの
   - テーマは1つに絞る（広すぎる場合は焦点を提案する）

2. **インタビュー開始**
   - 最初の質問を1つ投げかける
   - 質問はオープンエンド型にする（はい/いいえで終わらない）
   - 例: 「〇〇について、どういうきっかけでそう考えるようになりましたか？」

3. **深掘り（2〜4ラリー）**
   - オーナーの回答に対して、さらに深掘りする質問を1つずつ投げる
   - 以下の方向で掘り下げる:
     - **具体化**: 「具体的にはどんな経験がありましたか？」
     - **背景**: 「なぜそう考えるようになったのですか？」
     - **対比**: 「以前は違う考えでしたか？何が変わりましたか？」
     - **影響**: 「その考えは日常の行動にどう影響していますか？」
     - **教訓**: 「そこから得た教訓や原則はありますか？」
   - 無理に全方向を網羅しない。会話の流れに自然に従う
   - 合計3〜5ラリー程度でまとめに入る（長すぎない）

4. **要約と確認**
   - インタビューで得られた内容を要約してオーナーに見せる
   - 記録先のファイル名（ケバブケース）を提案する
   - 「この内容で thoughts/ に記録してよいですか？」と確認する

5. **記録**
   - `mcp__life-rag__search(query="記録内容の要約", layer="thoughts")` で類似の既存ファイルを検索し、新規作成か既存更新かを判定する
   - 承認を得たら `thoughts/` にファイルを作成または更新する
   - テンプレート `templates/thoughts.md` に従う
   - 当日の日記 `journal/` にもインタビューの概要を追記する
   - 変更をコミットする

## 質問設計のガイドライン

- **1回に1つだけ質問する**（複数同時に聞かない）
- **オーナーの言葉を拾って次の質問につなげる**（台本的にならない）
- **答えやすい質問から始める**（いきなり抽象的な問いを投げない）
- **沈黙や「わからない」も受け入れる**（別の角度から聞き直す）
- **誘導しない**（AIの推測を押し付けず、オーナー自身の言葉を引き出す）

## 記録のルール

- thoughts/ には洞察・方針・教訓を蒸留して記録する（具体的エピソードは省く）
- journal/ にはインタビューで話した内容の概要を時系列として記録する
- オーナーの表現や語彙をできるだけ活かす
- 既存の thoughts/ ファイルと関連がある場合はリンクで参照する
