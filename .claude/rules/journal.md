# 日記（具体層）の書き方ルール

## 基本原則

- オーナーの言葉・表現をできるだけそのまま記録する
- 文脈・時系列・感情のニュアンスを保持する
- 情報の重複を許容する（時系列の記録として価値がある）
- YAML frontmatter に date と tags を含める
- 既存の日記ファイルに追記する場合は既存内容を消さない
- 言語: 日本語

## イベント指向の構造

- 日記は `## イベント名` の見出しで区切り、出来事単位で記述する
- 各イベントには以下のサブ見出しを必要に応じて設ける:
  - `### 出来事` — 何があったか
  - `### 感情・内面` — そのとき何を感じたか
  - `### 気づき・学び` — そこから何に気づいたか
- リストの羅列ではなく、物語のように文脈を持たせて書く
- すべてのサブ見出しを埋める必要はない

## 情緒・内面へのフォーカス

- 事実の記録だけで終わらせず、感情・内面の動きを意識的に記録する
- 「なぜそう感じたか」まで掘り下げる
- 体調・気分・エネルギーレベルも情緒の一部として扱う
