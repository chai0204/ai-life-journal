# /recall — 今日を振り返る

AIが質問を投げかけることで、オーナーがその日の出来事・感情・気づきを思い出し、それを journal/ に記録する。

## 背景

日記を書きたい意志はあるが、「今日何があったっけ？」と具体的な内容が浮かばないことがある。白紙の前で手が止まる問題を、AIからの質問で解消する。`/interviewer` が特定テーマの思考を深掘りするのに対し、`/recall` は今日という一日を時間軸に沿って振り返ることに特化する。

## 手順

1. **準備**
   - 今日の日付を確認する
   - `journal/YYYY/MM/YYYY-MM-DD.md` が既に存在すれば読み込む（既に書かれた内容と重複しないようにする）
   - `mcp__life-rag__search(query="最近の出来事", layer="journal", date_from="直近数日")` で直近の日記を検索し、継続中のトピックや前日の流れを把握する
   - 曜日を把握する（平日か休日かで質問の方向が変わる）

2. **振り返り開始**
   - 最初の質問を1つ投げかける
   - 時間軸の早い方（朝）から始めるのが自然
   - 例: 「今日はどんな一日でしたか？朝はどう始まりましたか？」

3. **振り返りの展開（2〜4ラリー）**
   - オーナーの回答を受けて、まだ触れられていない時間帯・側面について質問する
   - 以下の軸で聞く:
     - **時間帯**: 「午後はどう過ごしましたか？」「夕方以降は？」
     - **仕事・活動**: 「今日取り組んだことで印象に残っていることはありますか？」
     - **人との関わり**: 「誰かと話したり、やり取りしたことはありましたか？」
     - **感情・内面**: 「そのときどんな気持ちでしたか？」「それを聞いてどう感じましたか？」
     - **体調・エネルギー**: 「今日の体調や気分はどうでしたか？」
     - **気づき・学び**: 「今日新しく気づいたことや学んだことはありますか？」
   - すべてを網羅する必要はない。会話の流れに自然に従う
   - **事実の回答に対して、感情面を1段掘り下げる質問を意識する**
     - 例:「〇〇をやったんですね。それをやっているときどんな気持ちでしたか？」
     - 例:「〇〇が大変だったとのことですが、具体的にどのあたりが辛かったですか？」
   - 合計3〜5ラリーでまとめに入る

4. **日記の下書きと確認**
   - 会話で得られた内容をもとに **イベント指向** で日記の下書きを作成する
   - 各イベントに `## イベント名` の見出しを立て、サブ見出し（出来事/感情・内面/気づき・学び）で構成する
   - オーナーに下書きを見せて確認する
   - 「この内容で journal/ に記録してよいですか？修正や追加があれば教えてください」

5. **記録**
   - 承認を得たら `journal/YYYY/MM/YYYY-MM-DD.md` に記録する
   - 既存ファイルがあれば追記、なければテンプレートから新規作成する
   - 会話の中でタスク・予定が浮上した場合は `gh issue create` で Issue を作成する
   - 変更をコミットする

6. **抽象層への提案**
   - 日記の内容に以下が含まれていれば、抽象層への記録も提案する:
     - 客観的な知識・学び → `knowledge/` へ
     - 主観的な考え・教訓 → `thoughts/` へ
   - 必ずオーナーに確認してから記録する

## 質問設計のガイドライン

- **1回に1つだけ質問する**（複数同時に聞かない）
- **時間軸に沿って自然に流れる**（朝→昼→夜の順が基本）
- **答えやすい具体的な質問にする**（「今日どうでしたか」より「朝はどう始まりましたか」）
- **オーナーの回答から次の質問を拾う**（台本的にならない）
- **「特にない」「普通だった」も受け入れる**（無理に掘り下げず次の軸へ移る）
- **誘導しない**（AIの推測を押し付けず、オーナー自身の言葉を引き出す）
- **事実→感情の掘り下げを意識する**: 出来事を聞いた後、それに対する感情・内面を自然に尋ねる
- **/interviewer との違いを意識する**: 深い思考の掘り下げではなく、今日の出来事の想起を助けることが目的

## 記録のルール

- journal/ にはオーナーの言葉・表現をできるだけそのまま活かす
- テンプレート `templates/journal.md` のイベント指向構造に従う
- 文脈・時系列・感情のニュアンスを保持する
- 既存の日記内容がある場合は消さずに追記する
